# Performance Benchmarking Suite: .NET Core vs. Laravel 🚀
### Isolated Automated Simulation (Vanilla vs. Optimized)

هذا المشروع عبارة عن بيئة اختبار أداء مؤتمتة بالكامل (Automated Benchmarking Suite) تقارن بين إطاري العمل **ASP.NET Core** و **Laravel**. تم تصميم المحاكاة لاختبار قدرة الأنظمة على الصمود تحت ضغط عالي جداً يصل إلى **10,000 مستخدم متزامن**، مع قياس كفاءة استهلاك الموارد (CPU & RAM) بدقة.

---

## ⚠️ متطلبات النظام الهامة (System Requirements)

لكي يعمل هذا المختبر بنجاح على بيئة Windows، يجب توفر المتطلبات التالية:

1. **Docker Desktop**: المحرك الأساسي لتشغيل الحاويات.
2. **WSL 2 (Windows Subsystem for Linux)**:
   > **ملاحظة حرجة:** يجب التأكد من تثبيت وتفعيل **WSL 2** على نظام ويندوز لديك وتحديثه إلى آخر إصدار. يعتمد Docker Desktop بشكل كلي على WSL كـ Backend لتشغيل حاويات Linux بكفاءة عالية. بدون WSL، لن تتمكن من تشغيل المحاكاة.
3. **Grafana k6**: لتنفيذ هجمات اختبار الجهد (Stress Testing).
4. **Node.js**: لمعالجة البيانات وتحديث لوحة التحكم التفاعلية.

## 📑 عن المشروع (About the Project)

يهدف المشروع إلى تقديم إجابة تقنية دقيقة حول أداء التقنيات المختلفة عند عزلها في بيئة حاويات (Docker). يتم الاختبار على أربع نسخ مختلفة تم رفع صورها مسبقاً على **Docker Hub** تحت حساب `mohammedalhady`:

* **Laravel Vanilla**: النسخة القياسية من إطار عمل لارافل.
* **Laravel Octane (Beast)**: النسخة المحسنة والمُسرَّعة باستخدام محرك عالي الأداء.
* **ASP.NET Core Vanilla**: النسخة القياسية من دوت نت.
* **ASP.NET Core Native AOT (Beast)**: النسخة المحسنة والمترجمة مسبقاً للغة الآلة لضمان أقصى سرعة استجابة.

## 📐 منهجية الاختبار (Methodology)

يعتمد هذا المختبر على معايير صارمة لضمان دقة النتائج وعدالة المقارنة:

* **تقييد الموارد:** تم تحديد الموارد لكل خدمة بـ (1 CPU) و (1GB RAM) لضمان بيئة اختبار موحدة.
* **العزل التام:** يتم تشغيل واختبار كل حاوية على حدة، ثم مسحها بالكامل قبل بدء الاختبار التالي لتجنب تداخل استهلاك الذاكرة.
* **مسار العمل المؤتمت:** يتم التحكم في الدورة الكاملة (تشغيل ➔ هجوم ➔ قياس ➔ إغلاق) عبر سكربت أتمتة واحد.
